# Stage 1 — Single Guidelines File

This is what the initial AI guidelines look like for `library-service`.
A single file, generated by asking the AI to read the codebase and capture what matters.

---

## How to generate this

Paste this prompt into Claude Code (or Junie, or Cursor) at the start of a new repo:

```
Read this codebase and write a CLAUDE.md that captures:
- How to build and run the project
- The tech stack and architecture (only what isn't obvious from the code)
- Coding conventions and non-obvious constraints used in this project
- Anything a developer would need to know that can't be inferred from reading the source

Keep it under 100 lines. Only include non-inferable information.
```

---

## Result: `CLAUDE.md` (what the AI generates)

```markdown
# library-service

## Build & Run
./gradlew clean build
./gradlew quarkusDev   # live reload on :8080

## Stack
Java 21 · Quarkus 3 · JAX-RS · CDI · Jackson · Gradle 8
Tests: JUnit 5 + Mockito + AssertJ

## Architecture
LibraryResource (JAX-RS) → LibraryService (CDI) → LibraryStore (ConcurrentHashMap)
Records: Book, Loan — immutable, use with*() helpers for state transitions.

## Coding Standards
- Java 21: records, Optional chains, text blocks, switch expressions
- NEVER use plain HashMap in @ApplicationScoped beans — always ConcurrentHashMap
- NEVER skip layers: Resource delegates to Service; Service uses Store
- Tests: Given-When-Then structure, no eq() matchers in Mockito
- New domain records must have a JSON deserialization test

## Known Gotchas
- Borrow availability flip: use ConcurrentHashMap.replace(key, expected, new) — NOT get-then-put.
  Plain put() after get() has a race condition under concurrent requests.
- JAX-RS path ordering: @Path("/search") must appear BEFORE @Path("/{id}") in the same class.
  If /{id} comes first, "search" is matched as an ID — no compile error, wrong runtime behavior.
- Records are immutable — do not add setters. Use Book.withAvailable() / Loan.withReturn().
```
